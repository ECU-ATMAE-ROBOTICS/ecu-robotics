Pull Request Guidelines

Formatting Requirements

When submitting a pull request, please adhere to the following formatting requirements:

  • Indentation: Use spaces instead of tabs for indentation. Set your text editor to use 2 spaces for indentation.
  • Line Length: Limit lines to a maximum of 80 characters to ensure readability.
  • Consistent Naming: Follow the project's naming conventions for variables, functions, and classes.
  • Code Comments: Include meaningful comments to explain complex sections of code or provide context.
  • Remove Debugging Code: Ensure that any temporary debugging code or console logs are removed before submitting the pull request.

Submitting a Pull Request

When submitting a pull request, please follow these steps:

  1. Fork the repository on GitHub.
  2. Create a new branch for your changes.
  3. Make your changes and commit them with descriptive commit messages.
  4. Push your branch to your forked repository.
  5. Open a pull request from your branch to the original repository.
  6. Provide a clear and concise title and description for your pull request.
  7. Include any relevant documentation or screenshots if applicable.
  8. Assign the pull request to the appropriate reviewer(s).

Code Review Process

Once you've submitted a pull request, the code review process begins. Here's what to expect:

  • Reviewers will provide feedback on your code, including suggestions for improvement.
  • Please be responsive to feedback and make the necessary changes to your code.
  • Discussion and collaboration are encouraged. Feel free to ask questions or seek clarification.
  • Once the code review is complete and all feedback has been addressed, your changes will be merged into the main branch.
