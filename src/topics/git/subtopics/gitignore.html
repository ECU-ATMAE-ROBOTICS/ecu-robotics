Gitignore

The .gitignore file is used to specify intentionally untracked files and directories that Git should ignore. These files and directories are typically generated or contain sensitive information that should not be tracked by Git.

Creating a Gitignore File

To create a .gitignore file, follow these steps:

  1. Create a new plain text file in the root directory of your Git repository.
  2. Name the file .gitignore (including the dot at the beginning).
  3. Add rules to the file specifying the files, directories, or patterns to be ignored.

Common Gitignore Rules

Here are some common patterns and rules used in a .gitignore file:

  • *.log: Ignore all files with the extension .log.
  • build/: Ignore the entire build directory.
  • secret.txt: Ignore a specific file named secret.txt.
  • temp*: Ignore all files or directories starting with temp.
  • !important.txt: Exclude a file named important.txt from being ignored.

Global Gitignore

You can also create a global .gitignore file that applies to all your Git repositories. This file is not stored in any repository but is associated with your Git configuration on your local machine.
