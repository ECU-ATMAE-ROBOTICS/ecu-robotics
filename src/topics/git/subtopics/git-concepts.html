Git Concepts

Git is built upon several key concepts that form the foundation of its functionality. Understanding these concepts is essential for effectively using Git.

Here are some important Git concepts:

  • Repository: A repository is a directory that contains all the files and version history of a project.
  • Commit: A commit is a record of changes made to the repository at a specific point in time.
  • Branch: A branch is a parallel line of development that allows independent work without affecting the main codebase.
  • Merge: Merging combines changes from one branch into another, integrating them together.
  • Remote: A remote is a shared repository hosted on a server, allowing collaboration with others.
  • Tag: A tag is a label assigned to a specific commit to mark significant points in the project's history.
