Git Basics

What is Git?

Git is a distributed version control system that allows multiple people to collaborate on a project. It tracks changes to files and enables users to manage different versions of their code.

Common Git Commands

  • git init: Initializes a new Git repository in the current directory.
  • git clone <repository-url>: Creates a local copy of a remote repository.
  • git add <file>: Adds a file or changes to the staging area.
  • git commit -m "commit message": Commits the changes in the staging area with a descriptive message.
  • git push <remote> <branch>: Pushes the committed changes to a remote repository.
  • git pull <remote> <branch>: Fetches and merges changes from a remote repository to the current branch.
  • git branch: Lists all branches in the repository.
  • git checkout <branch>: Switches to the specified branch.
  • git merge <branch>: Merges changes from the specified branch into the current branch.
  • git status: Shows the current status of the repository, including tracked/untracked files and branch information.

Basic Workflow

  1. Create or clone a repository using git init or git clone.
  2. Make changes to the files in your local repository.
  3. Add the changes using git add to include them in the next commit.
  4. Commit the changes using git commit with a meaningful message.
  5. Push the committed changes using git push to share them with others.
  6. Periodically pull changes using git pull to incorporate updates from the remote repository.

Conclusion

Git is a powerful version control system that helps you manage and track changes to your projects. By understanding the basics and using common Git commands, you can effectively collaborate with others and keep your codebase organized.
